# DB손해보험 보상청구 어드민 – 상세 화면 UI 명세

---

## 1. 화면 목적

이 화면은 DB손해보험 보상청구 어드민에서 단일 청구 건을 검토하기 위한 상세 페이지입니다.
한 화면에서 다음 정보를 빠르게 확인하고 보상 가능 여부를 판단할 수 있도록 구성했습니다.

- 청구인 정보
- 보상청구 내용
- 필수 첨부서류 업로드 현황 및 검증 결과
- 문서별 AI 추출데이터
- 보상청구 검토 결과 상태
- 검토자 안내/설명 문구

---

## 2. 전체 레이아웃 구조

app (1440px 고정 폭)
- sidebar : 좌측 내비게이션 (240px)
- main : 우측 메인 영역
  - notice : 상단 안내 배너
  - content : 3컬럼 그리드
    - 좌측 컬럼 : 청구인 정보 / 보상청구 내용
    - 중앙 컬럼 : 첨부서류 AI 검토 / AI 추출데이터
    - 우측 컬럼 : 보상청구 검토 결과 / 설명 영역

---

## 3. 주요 컴포넌트 요약

### 3.1 사이드바 (sidebar)

- 역할: 어드민 전체 내비게이션
- 배경색: #172b55 (인디고 블루)
- 메뉴 구조 예시
  - 대시보드
  - 보상 청구 (활성 메뉴)
- 활성 메뉴는 파란색(#2563eb) 배경 + 흰색 텍스트로 표시합니다.

### 3.2 상단 Notice 배너 (notice)

- 배경: #e8f2ff
- 테두리: #c3d5ff
- 내용: 청구양식/첨부서류 다운로드 안내 + AI 분석 안내 문구 두 줄
- 좌측에 NOTICE 라벨과 작은 파란 점 아이콘을 배치합니다.

### 3.3 공통 카드 (card)

- 배경: 흰색
- 테두리: #dde4f5
- 그림자: 0 10px 30px rgba(15, 23, 42, 0.08)
- 라운드: 16px
- card_header 안에 card_title 로 제목을 표시합니다.

### 3.4 청구인 정보 / 보상청구 내용

- 라벨/값 2컬럼 그리드 구조 (110px + 나머지)
- field_label: 회색(#9ca3af)
- field_value: 본문 색(#111827)

### 3.5 보상청구 신청 파일 카드 (file-card)

- 배경: #eef2ff
- 테두리: #c3d0f0
- 왼쪽: 파일 아이콘 + 제목 '보상청구 신청 파일'
- 오른쪽: 파란색 다운로드 버튼 (pill 형태)

### 3.6 첨부서류 AI 검토 테이블 (table)

- 컬럼: 준비서류 / 업로드 여부 / 다운로드 / 검증 여부
- 헤더 배경: #f3f4ff
- 상태값은 badge-status 스타일 사용 (연한 초록 배경, 진한 초록 텍스트)

### 3.7 AI 추출데이터 문서 카드 (doc-item)

- 문서 번호 뱃지, 문서 제목, 분류 신뢰도(유효) 표시
- 하단에 추출데이터 라벨과 key/value 리스트를 플렉스 그리드로 배치
- 카드 배경: #f9fafb, 테두리: #dde4f5

### 3.8 보상청구 검토 결과 카드 (result-card)

- 제목: '보상청구 검토 결과'
- 본문에 '보상청구 검토 : 보상청구 가능' 을 초록 뱃지로 표시
- 특약명, 한도, 자기부담금을 간략하게 텍스트로 정리

### 3.9 설명 패널 (description)

- 검증 결과 요약, 플랜/특약/한도 설명, 지급 판단 가이드 문구를 2개의 블록으로 구성
- 폰트 12px, 줄 간격 1.6, 중요한 단어는 굵게 표시

## 4. 스타일 가이드 요약

- Primary Blue: #2563eb
- Sidebar Indigo: #172b55
- Page Background: #f3f6fc
- 카드 제목: 15px / 굵게
- 본문 텍스트: 13px
- 설명 텍스트: 12px

---

## 5. 개발 연동 메모

- 현재 HTML은 예시 데이터 기준입니다.
- 실제 연동 시에는 서버에서 내려주는 JSON 데이터를 기반으로
  청구인 정보, 청구 내용, 문서 리스트, AI 추출데이터, 검토 결과를 바인딩합니다.
- 문서 리스트는 반복 렌더링으로 doc-item 을 생성하고,
  각 다운로드 버튼은 실제 파일 경로로 연결합니다.


## 6. 대시보드 CSS 클래스별 컴포넌트 명세

| 클래스명 | 소속 영역 | 역할 / 설계 의도 |
|---------|-----------|------------------|
| `.app` | 전체 레이아웃 | 1440px 고정 폭 레이아웃 컨테이너. 좌측 사이드바 + 우측 메인 영역을 flex로 배치. |
| `.sidebar` | 사이드바 | 좌측 내비게이션 컬럼. 배경 인디고(#172b55), 메뉴/로고 배치. |
| `.sidebar__logo` | 사이드바 | 서비스 로고 영역. 상단 여백 + 하단 마진으로 메뉴와 시각적 분리. |
| `.sidebar__section` | 사이드바 | 메뉴 리스트 래퍼. 추후 섹션 구분 시 재사용 가능. |
| `.sidebar__nav` | 사이드바 | 사이드 메뉴 UL. 리스트 점 제거, 수직 메뉴 컨테이너. |
| `.sidebar__item` | 사이드바 | 공통 메뉴 아이템 스타일. 패딩, radius(10px), hover 효과, 기본 텍스트 컬러/불투명도 정의. |
| `.sidebar__item--active` | 사이드바 | 현재 선택된 메뉴 상태. 파란 배경(#2563eb)에 흰 텍스트, 강조용. |
| `.main` | 메인 영역 | 우측 전체 콘텐츠 영역. 상하좌우 패딩과 column 레이아웃 담당. |
| `.notice` | 상단 배너 | 안내 문구 영역. 연한 파란 배경(#e8f2ff), 라운드, 테두리로 메인 내용과 구분되는 인포 배너. |
| `.notice__label` | 상단 배너 | “NOTICE” 라벨. 프라이머리 블루 텍스트 + 아이콘과 수평 정렬. |
| `.notice__dot` | 상단 배너 | 라벨 앞 동그란 점 아이콘. 상태/강조 시각 요소. |
| `.notice__text` | 상단 배너 | 실제 안내 문구가 들어가는 텍스트 컨테이너. p 태그 두 줄 사용. |
| `.content` | 메인 그리드 | 3컬럼 그리드 레이아웃. 좌/중/우 카드 영역을 고정 비율로 배치. |
| `.card` | 공통 카드 | 모든 정보 박스의 공통 UI. 흰 배경, 라운드(16px), 그림자, 테두리. |
| `.card__header` | 카드 | 카드 상단 헤더 영역. 제목/우측 액션 배치를 위한 flex 컨테이너. |
| `.card__title` | 카드 | 카드 제목 텍스트. 15px, 볼드, 아이콘과 함께 인라인 정렬 가능. |
| `.card__grid` | 카드 바디 | 라벨/값 2컬럼 레이아웃. 좌측 고정폭 라벨, 우측 값. |
| `.field-label` | 폼 필드 | 항목 라벨 텍스트. 보조색(#9ca3af)로 구분. |
| `.field-value` | 폼 필드 | 실제 데이터 값 텍스트. 기본 본문 컬러(#111827). |
| `.claim-section` | 좌측 컬럼 | 청구인 정보/보상청구 내용을 묶는 섹션 래퍼. 여유 마진 관리용. |
| `.claim-section__file` | 좌측 컬럼 | 보상청구 신청 파일 카드가 들어가는 영역. 본문과의 간격을 위한 상단 마진. |
| `.file-card` | 파일 카드 | 보상청구 신청 파일 박스 UI. 연보라 배경(#eef2ff), 라운드, 좌측 정보 + 우측 버튼 구조. |
| `.file-card__left` | 파일 카드 | 파일 아이콘 + 제목을 감싸는 좌측 영역. 수평 정렬. |
| `.file-card__icon` | 파일 카드 | 파일 아이콘 래퍼. 아이콘 정중앙 정렬, 크기 고정. |
| `.file-card__title` | 파일 카드 | 파일명 텍스트(“보상청구 신청 파일”). 13px, 600, 중간 톤 컬러. |

| `.table` | 중앙 테이블 | 첨부서류 AI 검토 테이블. 헤더/바디 스타일, 셀 padding/정렬 제어. |
| `.badge-status` | 상태 뱃지 | “예”, “유효” 등 상태를 표시하는 초록 pill 뱃지. 배경 연한 초록, 진한 초록 텍스트. |

| `.result-card` | 우측 결과 | “보상청구 검토 결과” 카드에 적용되는 강조용 클래스. 제목 컬러를 프라이머리 블루로 통일. |
| `.description` | 우측 설명 | 설명 텍스트 전체 컨테이너. 기본 폰트/줄간격 정의. |
| `.description__block` | 우측 설명 | 문단 단위 블록. 블록 간 점선 구분선 + 상단 여백. |
