Case Study

Problem
핵심 보장 정보보다 장식성 콘텐츠가 먼저 노출되어 이탈률이 높고 CTA 도달 전 이탈이 잦았습니다.
Goal
상담 CTA 클릭률 20% 개선, 핵심 정보 섹션 체류시간 증가를 목표로 구조를 재설계했습니다.
Research
동종 보험 랜딩 6개를 비교해 CTA 위치, 정보 계층, 신뢰 요소(리뷰/보장 요약) 배치를 분석했습니다.
Insight
사용자는 보험 상품 설명보다 “나에게 어떤 결과가 생기는가”를 먼저 확인할 때 전환 가능성이 높았습니다.
Solution
첫 화면에 문제-해결 메시지와 CTA를 고정 배치하고, 보장 요약 카드를 상단으로 이동해 스크롤 초반 의사결정을 돕도록 구성했습니다.
Why
CTA를 상단에 고정하면 클릭 동선이 짧아지고, 카드형 정보는 스캔 속도를 높여 비교 판단 시간을 줄일 수 있기 때문입니다.
Result
랜딩 흐름 상 클릭 동선이 단축되었고, 상담 CTA 노출 빈도가 증가하는 구조로 개선했습니다. (estimated)

Case Study

Problem
관심 아이템이 여러 플랫폼에 흩어져 저장되어, 실제 구매 시점에는 다시 찾지 못하는 문제가 반복되었습니다.
Goal
저장한 아이템의 재탐색 시간 단축, 비교/결정 흐름 단계를 3탭 이내로 단순화하는 것을 목표로 했습니다.
Research
주변 사용자 인터뷰/관찰에서 저장 방식은 다양하지만 “저장 후 재사용률”이 낮다는 공통 패턴을 확인했습니다.
Insight
핵심 문제는 저장 기능 부재가 아니라, 저장 후 분류/비교/재탐색 맥락이 사라지는 구조였습니다.
Solution
URL 저장 + 태그 분류 + 비교 모드를 중심으로 IA를 재설계하고, 저장 이유 메모를 강제하지 않고 자연스럽게 입력하게 구성했습니다.
Why
사용자가 원하는 건 기록 자체가 아니라 의사결정 보조이므로, 저장보다 비교/우선순위 판단 UI를 우선했습니다.
Result
탐색 플로우를 3회 개선하며 의사결정 전환 경로를 단순화했고, 프로토타입 테스트 기준으로 재탐색 혼란도를 낮췄습니다. (based on internal feedback)

Case Study

Problem
청구 건 처리 시 다수 화면 이동이 필요해 처리 속도가 느리고, 상태 우선순위 판단이 어려웠습니다.
Goal
페이지 이동 횟수 50% 이상 감소, 상태 기반 우선 처리 판단을 1화면에서 가능하게 만드는 것을 목표로 했습니다.
Research
운영 담당자의 반복 액션(상태 확인 → 상세 확인 → 처리 입력)을 기준으로 병목 구간을 정의했습니다.
Insight
운영자는 미관보다 “정보 스캔 속도”와 “컨텍스트 유지”를 더 중요하게 여겼습니다.
Solution
테이블 + 사이드 패널 구조로 전환하고, 상태 뱃지/필터를 전면 배치해 상세 페이지 이동 없이 처리하도록 설계했습니다.
Why
대시보드 업무는 연속 처리 과업이므로, 컨텍스트가 끊기면 처리 품질과 속도가 동시에 하락하기 때문입니다.
Result
페이지 이동 중심 플로우를 단일 화면 중심 플로우로 재구성해 처리 동선을 단축했습니다. (estimated)

Case Study

Problem
기존 페이지는 서비스 신뢰 메시지와 CTA 연결이 약해, 사용자가 결제 솔루션의 장점을 빠르게 이해하기 어려웠습니다.
Goal
핵심 가치 전달 시간을 단축하고, 문의/상담 CTA 전환 흐름을 강화하는 것이 목표였습니다.
Research
핀테크 랜딩 사례를 참고해 신뢰 요소, 기능 설명, CTA 배치 순서를 비교 분석했습니다.
Insight
핀테크 서비스는 시각 완성도보다 “신뢰 근거 + 명확한 다음 행동”이 전환에 더 큰 영향을 줍니다.
Solution
Midjourney 기반 키비주얼을 제작해 첫 인지 품질을 높이고, 가치 제안/근거/CTA가 자연스럽게 이어지도록 랜딩 구조를 재배치했습니다.
Why
신뢰가 선행되지 않으면 클릭이 일어나지 않기 때문에, 비주얼-카피-CTA의 정보 흐름을 단일 서사로 설계했습니다.
Result
브랜드 신뢰 메시지와 CTA 노출 구조가 강화되어 문의 전환 가능성이 높은 랜딩 흐름을 확보했습니다. (estimated)
