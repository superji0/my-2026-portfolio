# 픽클리 앱 (Pickly App)

## Overview

* **Project:** 픽클리 — 개인 앱 서비스 기획 및 UX/UI 디자인
* **Role:** 서비스 기획 + UX 리서치 + UI 디자인 + 프로토타이핑 (단독)
* **Duration:** 2024 — 2025
* **Summary:** 사용자가 일상에서 마음에 드는 아이템을 저장·분류하고, 구매 결정을 도와주는 큐레이션 앱. 기획 제로 베이스에서 최종 UI까지 전 과정 단독 진행.

---

## Problem

원하는 제품을 여러 플랫폼에 흩어진 채로 저장하다 보니 구매 결정 시점에 다시 찾는 데 시간이 걸리고, 막상 찾아도 "이게 왜 좋았지?" 맥락을 잊어버리는 경험이 반복됨.

---

## User Insight

* 사용자들은 마음에 든 아이템을 **스크린샷, 카카오톡 나에게 보내기, 브라우저 북마크** 등 제각각으로 저장
* 저장 당시의 맥락(이유, 가격, 비교 대상)이 없으면 나중에 의미가 없음
* "나중에 살게" 버킷이 아닌, **지금 결정할 수 있게 돕는 도구** 가 필요함

---

## Analysis

* 기존 서비스(핀터레스트, 위시리스트 앱)는 수집 중심 → 결정 단계에서 도움 부족
* 핵심 차별점: 아이템 저장 + **비교/우선순위 판단** 기능 결합
* 사용자 플로우: 발견 → 저장 → 분류 → 비교 → 결정

---

## Solution

* 아이템 저장 시 **메모(이유) + 카테고리 태그** 즉시 입력 유도
* 저장 목록에서 **나란히 비교(Compare)** 모드 제공
* 우선순위 스와이프(Swipe to rank) 인터랙션으로 결정 피로 감소
* 홈 화면: "오늘 결정할 것" 추천 카드 노출

---

## UI Design

> 주요 화면: 홈 / 아이템 저장 / 카테고리 / 비교 모드 / 상세

* 스크린샷: `./images/kybo_screen.png`, `./images/kybo_screen02.png`, `./images/kybo_screen03.png`
* 모바일 퍼스트 설계: 엄지 동선 기반 하단 내비게이션
* 카드형 UI로 시각적 스캔 최적화
* 저장 → 분류 → 비교까지 3탭 이하 접근 가능 구조

---

## Before / After

| Before (기존 행동) | After (픽클리) |
|---------------|------------|
| 스크린샷 + 메모앱에 흩어서 저장 | 한 곳에 맥락 포함 저장 |
| 비교할 때 탭 여러 개 띄워놓음 | 앱 내 나란히 비교 모드 |
| 결정 시점에 맥락 소실 | 저장 시 메모로 결정 근거 유지 |

---

## Result

* 개인 프로젝트로, 실제 배포 전 단계 (프로토타입 완성)
* UX 리서치 → IA 설계 → 와이어프레임 → 최종 UI 전 과정 단독 완수
* Figma 프로토타입으로 사용자 테스트 1회 진행, 주요 플로우 개선 반영

---

## Learned

* **기획자 없이 디자이너가 직접 서비스를 설계**하는 경험 → 제품 오너십 향상
* "기능을 추가하는 것"보다 "불필요한 결정을 줄이는 것"이 UX의 핵심임을 체감
* IA 설계 단계에서 충분히 고민하지 않으면 UI 단계에서 구조적 수정이 반복됨
