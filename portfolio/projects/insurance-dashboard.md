# 보험 청구 관리 대시보드

## Overview

* **Project:** 보험 보상 청구 관리 어드민 대시보드
* **Role:** UI/UX Design + Web Publishing (단독)
* **Duration:** 2024
* **Summary:** 보험 보상 청구 처리 업무를 담당하는 내부 운영팀을 위한 어드민 시스템. 복잡한 청구 데이터를 한눈에 파악하고 빠르게 처리할 수 있는 인터페이스 설계 및 구현.

---

## Problem

운영팀이 청구 건을 처리할 때 여러 화면을 오가야 해서 처리 속도가 느리고, 청구 상태를 한눈에 파악하기 어려웠음. 기존 시스템은 테이블 나열 방식으로 우선순위 판단이 불가능했고, 필터/검색 기능이 없어 특정 건을 찾는 데 시간이 소요됨.

---

## User Insight

* 운영 담당자는 하루 평균 50+ 건을 처리함 → **속도와 스캔 가능성(Scannability)** 이 핵심
* 청구 상태(접수/검토/처리/완료)를 색상과 뱃지로 즉시 구분할 수 있어야 함
* 상세 정보 확인을 위해 페이지를 이동하면 컨텍스트가 끊겨 업무 효율 저하

---

## Analysis

* 기존 시스템: 단순 테이블 + 건별 페이지 이동 방식
* 가장 빈번한 액션: 상태 확인 → 상세 내용 열람 → 처리/코멘트 입력
* 데이터 밀도가 높아 카드형보다 **테이블 + 사이드 패널** 구조가 최적

---

## Solution

* 좌측 사이드바 내비게이션으로 카테고리별 빠른 이동
* 청구 목록 테이블에 상태 뱃지, 우선순위 컬러링 적용
* 행 클릭 시 우측 슬라이드 패널로 상세 확인 (페이지 이동 없음)
* 상단 필터 바로 상태/날짜/담당자별 즉시 필터링

---

## UI Design

> 스크린샷: `./images/mok.jpg`, `./images/mok.png`

* 다크 네이비 사이드바 + 화이트 컨텐츠 영역으로 시각적 위계 확립
* CSS Variables 기반 디자인 토큰으로 일관된 컬러 시스템 적용
* 카드형 KPI 위젯으로 전체 현황 스캔 가능

---

## Before / After

| Before | After |
|--------|-------|
| 단순 테이블, 상태 구분 없음 | 상태 뱃지 + 우선순위 컬러링 |
| 상세 보려면 페이지 이동 | 사이드 패널로 컨텍스트 유지 |
| 필터 없음 | 다중 조건 필터 즉시 적용 |

---

## Result

* 단일 화면에서 청구 현황 파악 → 페이지 이동 횟수 **약 60% 감소** (추정)
* 상태별 필터링으로 우선 처리 건 즉시 식별 가능
* Figma → HTML/CSS/JS 구현 기여도 100%

---

## Learned

* 어드민 UI는 "예쁨"보다 **정보 밀도 + 스캔 속도**가 우선
* 상태를 시각화하는 것만으로도 인지 부하를 크게 줄일 수 있음
* 실무 기획자/개발자와의 협업에서 UI가 커뮤니케이션 도구임을 체감

---
